How can my client send documents to me securely without having to sign up for an account?

HMAC generates a signature from a message. Only those who have the secret can generate the correct signature for the message. This who have message and the signature cannot figure
out what the secret is nor can they modify the message so that signature is still valid. There are no known security flaws or attacks on HMAC. We also include a decay, so the signature is
only valid for a limited period of time. Additionally, as soon as you mark a request as completed, no one except for the project team can access it.
